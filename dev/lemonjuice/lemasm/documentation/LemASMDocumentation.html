LemASM Documentation

The following is the documentation for the LemASM language.

Interpreter Arguments

Argument Result
-d Debug: Starts the interpreter in debug mode.
-h Help: Opens the Documentation HTML file.
-o <output_file> Output File: Speficies a file to output to.

Structure of a LemASM Program

A LemASM program is split into two sections: the Data Section and the Code Section.
The Data Section is used to define string constants before the code is executed.
The Data Section is optional, but must be placed before the Code Section.
The Code Section is where the actual code is written.
The following will show which instructions are available in each section.

How To Call A Section

Mnemonic/Symbol Result
#DATA Enters the Data Section.
#CODE Enters the Code Section.

Viable In Both Sections

Mnemonic/Symbol Result
// Comments the rest of the line.

The Data Section

Mnemonic/Symbol Result
STR <string_name> = "<string_text>" Creates a string with the name provided.

The Code Section

Mnemonic/Symbol Result
.<label_name> Defines a label.
ADD Adds the top two values of the stack, if two are avaible to add.
CPK Console Peek and Print: Peeks at the top value of the stack and prints it to the console, if there is a value.
CPP Console Pop and Print: Pops the top value off the stack and prints it to the console, if there is a value.
CPR Console Print: Prints a string to the console.
DIV Divides the top two values of the stack, if two are avaible to divide.
DUP Duplicates the top value of the stack, if one is available to duplicate.
JEQ <label_name> Jump Equal: Jumps to a specified label if the top two values of the stack are equal.
JGT <label_name> Jump Greater Than: Jumps to a specified label if the top value of the stack is greater than the second.
JLT <label_name> Jump Less Than: Jumps to a specified label if the top value of the stack is less than the second.
JMP <label_name> Jumps to a specified label.
JNQ <label_name> Jump Not Equal: Jumps to a specified label if the top two values of the stack not are equal.
MOD Mods the top two values of the stack, if two are avaible to mod.
MUL Multiplies the top two values of the stack, if two are avaible to multiply.
PSH <integer> Pushes a specified integer onto the stack.
POP Pops the top value of the stack, if there is a value that can be popped.
RAN Pushes a random integer onto the stack.
RET Returns the top number on the stack.
ROR Randomize Order: Randomizes the order of the stack.
SUB Subtracts the top two values of the stack, if two are avaible to subtract.
SWP Swaps the top two values of the stack, if two are avaible to swap.

Examples

Here are some examples of LemASM code.

Example 1: Hello World

hello_world.lemasm

#DATA
  STR hello_world = "Hello, World!"

  #CODE
  CPR hello_world
  RET

Output

Hello, World!
